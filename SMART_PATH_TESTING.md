# 智能路径处理功能测试指南

## 功能概述

新的智能路径处理功能会在每次解压完成后立即执行，而不是等到所有任务完成后再统一处理。该功能支持两种处理模式：

1. **拼接模式**：将父文件夹和子文件夹名称用下划线连接
2. **智能选择模式**：根据文件夹名长度和是否包含日文自动选择最合适的名称

## 配置方法

1. 打开软件，点击"设置"按钮
2. 在"高级设置"部分，勾选"启用智能路径处理"
3. 选择处理模式：
   - 拼接文件夹名字
   - 智能选择（优先选择长的、有日文的）
4. 确保输出模式设置为"输出到压缩文件同名文件夹"
5. 点击"保存配置"

## 测试场景

### 场景1：单层嵌套文件夹
```
压缩包: test.zip
解压后结构:
  test/
    └── content_folder/
        ├── file1.txt
        └── file2.txt
```

**预期结果**：
- 如果使用拼接模式：`test_content_folder`
- 如果使用智能选择模式：根据名称长度和日文情况选择

### 场景2：包含日文的文件夹
```
压缩包: archive.zip
解压后结构:
  archive/
    └── ドキュメント_非常に長い名前/
        └── data.txt
```

**预期结果**：
- 智能选择模式会优先选择包含日文的文件夹名

### 场景3：多个子文件夹（不处理）
```
压缩包: multi.zip
解压后结构:
  multi/
    ├── folder1/
    └── folder2/
```

**预期结果**：
- 不进行智能路径处理，保持原结构

### 场景4：直接包含文件（不处理）
```
压缩包: files.zip
解压后结构:
  files/
    ├── file1.txt
    └── file2.txt
```

**预期结果**：
- 不进行智能路径处理，保持原结构

## 处理流程

1. 解压完成后立即检查输出目录
2. 判断是否只有一个子文件夹且没有文件
3. 如果符合条件，根据选择的模式确定最终文件夹名
4. 生成临时名称（GUID前缀 + 文件夹名）
5. 移动子文件夹到上级目录
6. 删除空的原目录
7. 重命名去掉GUID前缀

## 日志信息

在处理过程中，会在日志中显示详细信息：
- `[智能路径] 开始处理: ...`
- `[智能路径] 当前目录: ... - 子文件夹数: ..., 文件数: ...`
- `[智能路径] 找到唯一子文件夹: ...`
- `[智能路径] 使用拼接模式/智能选择模式: ...`
- `[智能路径] 步骤1: 移动子文件夹到上级目录`
- `[智能路径] 步骤2: 删除空文件夹`
- `[智能路径] 步骤3: 重命名（去掉临时前缀）`
- `[智能路径] ✓ 智能路径处理完成`

## 注意事项

1. 只有在输出模式为"输出到压缩文件同名文件夹"时才会启用智能路径处理
2. 处理是异步进行的，不会阻塞主线程
3. 如果处理失败，会保留原始结构并记录错误日志
4. 临时文件名使用8位GUID前缀，避免命名冲突
