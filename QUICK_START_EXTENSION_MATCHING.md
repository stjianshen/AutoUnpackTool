# 扩展名匹配功能 - 快速开始

## 🎯 功能简介

扩展名匹配功能让你可以精确控制哪些文件会被检测为压缩文件。系统采用三级过滤机制：

```
1. 排除列表 → 直接跳过（最快）
2. 包含列表 → 魔术数验证（可靠）
3. 纯魔术数检测 → 兜底方案（全面）
```

## ⚙️ 配置步骤

### 第 1 步：打开设置
点击主窗口的 **"软件设置"** 按钮

### 第 2 步：找到扩展名设置区域
在设置对话框底部，你会看到：
```
压缩文件扩展名检测:
┌─────────────────────────────────────┐
│ 需要检测的扩展名（逗号分隔）:        │
│ [文本框]                             │
│                                     │
│ 需要排除的扩展名（逗号分隔）:        │
│ [文本框]                             │
│                                     │
│ 提示：符合扩展名的文件会用魔术数...  │
└─────────────────────────────────────┘
```

### 第 3 步：编辑扩展名列表

#### 默认值（推荐保留）
**需要检测的扩展名：**
```
.zip,.rar,.7z,.tar,.gz,.bz2,.xz,.zst,.zstd,.tgz,.tbz2,.tbz,.txz,.cab,.iso,.wim,.arj,.lzh,.cpio,.rpm,.deb
```

**需要排除的扩展名：**
```
.exe,.dll,.msi,.doc,.docx,.xls,.xlsx,.ppt,.pptx,.pdf,.jpg,.jpeg,.png,.gif,.bmp,.mp3,.mp4,.avi,.mkv
```

### 第 4 步：保存配置
点击 **"💾 保存配置"** 按钮

## 📋 常见配置场景

### 场景 1：只检测主流格式（ZIP/RAR/7Z）
```
需要检测的扩展名：.zip,.rar,.7z
需要排除的扩展名：.tar,.gz,.bz2,.xz,.zst,.zstd,.tgz,.cab,.iso,.wim,.arj,.lzh,.cpio,.rpm,.deb
```
**优点**：速度快，只处理最常用的格式

### 场景 2：排除所有文档和媒体文件
```
需要检测的扩展名：（保持默认）
需要排除的扩展名：.doc,.docx,.xls,.xlsx,.ppt,.pptx,.pdf,.jpg,.png,.gif,.bmp,.mp3,.mp4,.avi,.mkv,.wmv,.flv
```
**优点**：防止误检测伪装成压缩文件的文档或媒体文件

### 场景 3：添加自定义扩展名
如果你的压缩文件使用特殊扩展名（如 `.game`、`.data`）：
```
需要检测的扩展名：.zip,.rar,.7z,.tar,.gz,.game,.data
需要排除的扩展名：（保持默认）
```
**优点**：支持非标准扩展名的压缩文件

### 场景 4：严格模式（仅魔术数检测）
```
需要检测的扩展名：（留空）
需要排除的扩展名：.exe,.dll,.msi,.bat,.cmd,.ps1
```
**优点**：不依赖扩展名，完全通过文件头特征识别，最可靠

### 示例 1：正常 ZIP 文件
```
拖入文件：archive.zip

```

**日志输出：**

**日志输出：**


## 💡 实用技巧

### 技巧 1：提高检测速度
将不需要的格式添加到排除列表，避免不必要的魔术数检测：
```
需要排除的扩展名：.tar,.gz,.bz2,.xz（如果你不需要这些格式）
```

### 技巧 2：防止误检测
如果有大量非压缩文件被误识别，将它们添加到排除列表：
```
需要排除的扩展名：.dat,.bin,.pkg,.pak（游戏数据文件常见扩展名）
```

### 技巧 3：调试检测问题
查看日志输出，了解每个文件的检测过程：
- `[跳过]` → 在排除列表中
- `[检测] ... 扩展名匹配` → 在包含列表中，正在验证
- `[确认]` → 魔术数验证通过
- `[拒绝]` → 魔术数验证失败
- `[检测] ... 扩展名不匹配` → 不在列表中，直接魔术数检测

### 技巧 4：恢复默认设置
如果想恢复默认配置：
1. 清空两个文本框
2. 点击保存
3. 重新打开设置，会显示默认值

## ⚠️ 注意事项

1. **排除列表优先级最高**
   - 即使 `.zip` 同时在两个列表中，也会被排除
   - 设计目的：防止误检测重要文件

2. **魔术数验证更可靠**
   - 扩展名可以被轻易修改
   - 魔术数是文件的真实特征，难以伪造

3. **分卷压缩包例外**
   - `.001`, `.part1.rar` 等分卷格式不受扩展名列表限制
   - 系统会自动检测并处理

4. **复合扩展名支持**
   - `.tar.gz`, `.tar.zst` 等双层扩展名会被正确识别
   - 无需特殊配置

5. **性能优化**
   - 排除列表中的文件：不读取文件内容，最快
   - 包含列表 + 验证：读取文件头（最多64字节），较快
   - 纯魔术数检测：读取文件头，较慢但更全面

## 📊 支持的魔术数类型

| 格式 | 魔术数 | 说明 |
|------|--------|------|
| ZIP | `50 4B 03 04` | 标准 ZIP 压缩包 |
| RAR 5 | `52 61 72 21 1A 07 01 00` | RAR 5.0+ |
| RAR 1.5 | `52 61 72 21 1A 07 00` | RAR 旧版本 |
| 7Z | `37 7A BC AF 27 1C` | 7-Zip |
| GZ | `1F 8B` | GZip |
| BZ2 | `42 5A 68` | BZip2 |
| XZ | `FD 37 7A 58 5A 00` | XZ |
| ZSTD | `28 B5 2F FD` | Zstandard |
| CAB | `4D 53 43 46` | Microsoft Cabinet |
| ARJ | `60 EA` | ARJ |
| TAR | `ustar` (偏移257) | Tar 归档 |

## 🆘 故障排除

### 问题 1：某些压缩文件没有被检测到
**可能原因：**
- 扩展名在排除列表中
- 扩展名不在包含列表中且魔术数不匹配

**解决方法：**
1. 查看日志，确认检测状态
2. 从排除列表中移除该扩展名
3. 添加到包含列表
4. 如果魔术数不支持，考虑更新程序

### 问题 2：非压缩文件被误识别
**可能原因：**
- 文件恰好有压缩文件的魔术数（极少见）

**解决方法：**
1. 将扩展名添加到排除列表
2. 这是最快的解决方案

### 问题 3：想临时禁用扩展名检测
**解决方法：**
1. 清空"需要检测的扩展名"文本框
2. 保存配置
3. 系统会直接使用魔术数检测所有文件

### 问题 4：配置文件损坏
**解决方法：**
1. 删除 `%APPDATA%\AutoUnpackTool\settings.json`
2. 重新启动程序
3. 会自动创建新配置文件并使用默认值

## 📝 配置文件位置

```
%APPDATA%\AutoUnpackTool\settings.json
```

对应字段：
```json
{
  "ArchiveExtensions": ".zip,.rar,.7z,...",
  "ExcludedExtensions": ".exe,.dll,.msi,..."
}
```
