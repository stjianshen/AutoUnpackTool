# 智能路径处理功能更新日志

## 版本: 2026-04-12

### 新增功能

#### 1. 实时智能路径处理
- **改进前**：智能路径处理在所有任务完成后统一执行
- **改进后**：每次解压完成后立即执行智能路径处理
- **优势**：
  - 更快的响应速度
  - 更好的用户体验
  - 减少内存占用（不需要记录待处理目录）

#### 2. 多种处理模式
新增两种智能路径处理模式：

##### 拼接模式 (Concatenate)
- 将父文件夹和子文件夹名称用下划线连接
- 示例：`test` + `content` → `test_content`
- 适用场景：需要保留完整路径信息

##### 智能选择模式 (SmartSelect) - 默认
- 根据文件夹名长度和是否包含日文自动选择最合适的名称
- 评分系统：
  - 包含日文：+100分
  - 名称长度 > 10：+50分
  - 评分相同时选择较长的名称
- 适用场景：处理多语言文件，特别是日文内容

### 技术改进

#### 1. 简化的判断逻辑
- **改进前**：递归查找深层嵌套文件夹
- **改进后**：只判断一层（当前目录下是否只有一个子文件夹）
- **优势**：
  - 更简单的逻辑
  - 更快的执行速度
  - 更少的错误可能性

#### 2. 代码优化
- 移除了 `_pendingSmartPathDirs` 字典，不再需要记录待处理目录
- 简化了 `ProcessSmartPath` 方法签名（从2个参数减少到1个）
- 删除了全局统一处理的代码块

#### 3. 更好的错误处理
- 重命名失败时保留临时名称，避免数据丢失
- 详细的日志输出，便于调试和问题追踪

### UI 改进

#### 设置对话框
- 在"高级设置"中添加了智能路径处理模式选择
- 只有启用智能路径处理时才显示模式选择控件
- 提供了清晰的模式说明和使用建议

### 配置变更

#### AppSettings.cs
新增枚举类型：
```csharp
public enum SmartPathMode
{
    Concatenate,        // 拼接文件夹名字
    SmartSelect         // 智能选择（优先选择长的、有日文的）
}
```

新增配置属性：
```csharp
public SmartPathMode SmartPathProcessingMode { get; set; } = SmartPathMode.SmartSelect;
```

### 兼容性

- ✅ 向后兼容：如果用户之前启用了智能路径处理，默认使用智能选择模式
- ✅ 配置文件自动升级：旧配置文件会自动添加新的配置项
- ⚠️ 注意：只有在输出模式为"输出到压缩文件同名文件夹"时才会启用智能路径处理

### 测试建议

请参考 `SMART_PATH_TESTING.md` 文档进行功能测试。

### 已知限制

1. 只在输出模式为"输出到压缩文件同名文件夹"时生效
2. 只处理单层嵌套（当前目录下只有一个子文件夹）
3. 如果目标文件夹已存在，会先删除再移动

### 未来改进方向

1. 支持多层嵌套的智能路径处理
2. 添加自定义规则引擎
3. 支持更多的语言检测（中文、韩文等）
4. 提供预览功能，让用户确认处理结果
