# 文件夹拖放功能测试指南

## 功能概述

现在支持将文件夹直接拖放到应用程序窗口中，程序会自动：
1. 递归扫描文件夹及其子文件夹中的所有文件
2. 识别压缩文件（通过扩展名和魔术数验证）
3. 自动检测并合并分卷压缩文件
4. 将识别到的压缩文件添加到待处理列表

## 测试场景

### 场景 1：普通文件夹拖放
**测试步骤：**
1. 创建一个包含多个压缩文件的文件夹（如 .zip, .rar, .7z 等）
2. 将整个文件夹拖放到应用程序窗口
3. 观察日志输出

**预期结果：**
- 日志显示"检测到文件夹: [路径]"
- 日志显示扫描到的文件数量
- 每个压缩文件都经过魔术数验证
- 所有压缩文件被添加到待处理列表

### 场景 2：分卷压缩文件夹
**测试步骤：**
1. 创建一个包含分卷压缩文件的文件夹，例如：
   - archive.7z.001
   - archive.7z.002
   - archive.7z.003
2. 将文件夹拖放到应用程序窗口

**预期结果：**
- 日志显示"检测到分卷压缩包: 3 个分卷"
- 只添加一个条目到列表（主卷）
- 状态显示"等待处理 (分卷: 3)"
- 文件大小是所有分卷的总和

### 场景 3：混合内容文件夹
**测试步骤：**
1. 创建包含以下内容的文件夹：
   - 普通压缩文件（.zip, .rar）
   - 分卷压缩文件（.7z.001, .7z.002）
   - 非压缩文件（.txt, .jpg 等）
   - 子文件夹中包含更多压缩文件
2. 拖放文件夹到应用程序

**预期结果：**
- 只识别和添加压缩文件
- 非压缩文件被忽略
- 子文件夹中的压缩文件也被扫描到
- 分卷压缩被正确合并

### 场景 4：深层嵌套文件夹
**测试步骤：**
1. 创建多层嵌套的文件夹结构
2. 在不同层级放置压缩文件
3. 拖放根文件夹到应用程序

**预期结果：**
- 递归扫描所有子文件夹
- 找到所有层级的压缩文件
- 正确处理每个目录中的分卷压缩

## 注意事项

1. **分卷检测范围**：分卷文件必须在同一个目录下才能被正确识别和合并
2. **性能考虑**：对于包含大量文件的文件夹，扫描可能需要一些时间
3. **黑名单过滤**：仍然会应用黑名单规则，匹配的文件会被删除
4. **重复检测**：如果文件已经在列表中，不会重复添加

## 已知限制

1. 分卷文件必须位于同一目录中
2. 不支持跨文件夹的分卷合并
3. 大型文件夹扫描可能较慢（未来可优化为异步扫描）

## 故障排除

**问题：文件夹拖放后没有任何反应**
- 检查文件夹中是否有压缩文件
- 查看日志输出，确认是否扫描到文件
- 确认文件扩展名在支持的列表中

**问题：分卷压缩没有被正确识别**
- 确认所有分卷文件都在同一目录中
- 检查分卷命名是否符合标准格式（.7z.001, .part1.rar 等）
- 查看日志中的分卷检测信息

**问题：扫描速度很慢**
- 文件夹中文件数量过多
- 可以考虑优化：只扫描特定扩展名的文件
- 未来版本可以添加进度条显示
