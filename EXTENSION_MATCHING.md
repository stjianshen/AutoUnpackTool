# 扩展名匹配功能说明

## 功能概述

新增的扩展名匹配功能允许用户自定义哪些文件扩展名应该被检测为压缩文件，以及哪些扩展名应该被排除。系统会按照以下流程处理文件：

1. **排除列表检查**：如果文件扩展名在排除列表中，直接跳过，不进行任何检测
2. **包含列表检查**：如果文件扩展名在包含列表中，使用魔术数（文件头特征）验证是否为真正的压缩文件
3. **魔术数检测**：如果扩展名不在包含列表中，直接使用魔术数检测是否为压缩文件

## 配置方法

### 1. 打开设置对话框

在主窗口中点击"软件设置"按钮。

### 2. 配置扩展名列表

在设置对话框中找到"压缩文件扩展名检测"部分：

#### 需要检测的扩展名
- 这里列出所有可能需要检测的压缩文件扩展名
- 多个扩展名用逗号分隔
- 可以带点号（.zip）或不带点号（zip），系统会自动处理
- 默认值：`.zip,.rar,.7z,.tar,.gz,.bz2,.xz,.zst,.zstd,.tgz,.tbz2,.tbz,.txz,.cab,.iso,.wim,.arj,.lzh,.cpio,.rpm,.deb`

#### 需要排除的扩展名
- 这里列出的扩展名会被直接跳过，不会进行任何检测
- **优先级高于包含列表**：即使某个扩展名同时在两个列表中，也会被排除
- 多个扩展名用逗号分隔
- 默认值：`.exe,.dll,.msi,.doc,.docx,.xls,.xlsx,.ppt,.pptx,.pdf,.jpg,.jpeg,.png,.gif,.bmp,.mp3,.mp4,.avi,.mkv`

### 3. 保存配置

点击"💾 保存配置"按钮保存设置。

## 工作流程示例

### 示例 1：排除列表优先
```
文件：document.exe
扩展名：.exe（在排除列表中）
结果：直接跳过，不进行魔术数检测
日志：[跳过] document.exe: 扩展名在排除列表中
```

### 示例 2：包含列表 + 魔术数验证成功
```
文件：archive.zip
扩展名：.zip（在包含列表中）
魔术数：50 4B 03 04（ZIP文件头）
结果：确认为压缩文件，开始解压流程
日志：
  [检测] archive.zip: 扩展名匹配，进行魔术数验证...
  [确认] archive.zip: 魔术数验证通过，是压缩文件
```

### 示例 3：包含列表 + 魔术数验证失败
```
文件：fake.rar（实际是文本文件）
扩展名：.rar（在包含列表中）
魔术数：不匹配RAR文件头
结果：拒绝作为压缩文件处理
日志：
  [检测] fake.rar: 扩展名匹配，进行魔术数验证...
  [拒绝] fake.rar: 魔术数验证失败，不是压缩文件
```

### 示例 4：扩展名不在列表中，但魔术数匹配
```
文件：unknown_file
扩展名：无或未知
魔术数：37 7A BC AF 27 1C（7z文件头）
结果：确认为压缩文件
日志：
  [检测] unknown_file: 扩展名不匹配，尝试魔术数检测...
  [确认] unknown_file: 魔术数检测通过，是压缩文件
```

## 支持的魔术数类型

系统支持检测以下压缩格式的魔术数：

| 格式 | 魔术数（十六进制） | 说明 |
|------|-------------------|------|
| ZIP | 50 4B 03 04 / 50 4B 05 06 / 50 4B 07 08 | ZIP压缩包 |
| RAR 5.0 | 52 61 72 21 1A 07 01 00 | RAR 5.0及以上版本 |
| RAR 1.5 | 52 61 72 21 1A 07 00 | RAR 1.5旧版本 |
| 7Z | 37 7A BC AF 27 1C | 7-Zip压缩包 |
| GZ | 1F 8B | GZip压缩 |
| BZ2 | 42 5A 68 | BZip2压缩 |
| XZ | FD 37 7A 58 5A 00 | XZ压缩 |
| ZSTD | 28 B5 2F FD | Zstandard压缩 |
| CAB | 4D 53 43 46 | Microsoft Cabinet |
| ARJ | 60 EA | ARJ压缩 |
| TAR | "ustar" (偏移257字节) | Tar归档 |

## 常见使用场景

### 场景 1：只检测特定格式
如果你只想检测 ZIP、RAR 和 7Z 格式：
```
需要检测的扩展名：.zip,.rar,.7z
需要排除的扩展名：.tar,.gz,.bz2,.xz,.zst,.zstd,.tgz,.cab,.iso,.wim,.arj,.lzh,.cpio,.rpm,.deb
```

### 场景 2：排除可执行文件
防止误检测伪装的压缩文件：
```
需要排除的扩展名：.exe,.dll,.msi,.bat,.cmd,.ps1,.vbs,.js
```

### 场景 3：添加自定义扩展名
如果你的压缩文件使用特殊扩展名：
```
需要检测的扩展名：.zip,.rar,.7z,.custom,.mypack
```

## 注意事项

1. **排除列表优先级最高**：即使扩展名同时出现在两个列表中，也会被排除
2. **魔术数验证更可靠**：仅凭扩展名可能被欺骗，魔术数验证能确保文件确实是压缩格式
3. **性能考虑**：排除列表中的文件会快速跳过，不进行文件读取操作，性能最优
4. **分卷压缩包**：分卷格式（如 .001, .part1.rar 等）不受扩展名列表限制，会自动检测
5. **复合扩展名**：支持 .tar.gz、.tar.zst 等双层扩展名

## 配置文件位置

所有设置保存在：
```
%APPDATA%\AutoUnpackTool\settings.json
```

对应的配置项：
```json
{
  "ArchiveExtensions": ".zip,.rar,.7z,...",
  "ExcludedExtensions": ".exe,.dll,.msi,..."
}
```

## 故障排除

### 问题：某些压缩文件没有被检测到
**解决方案**：
1. 检查文件扩展名是否在排除列表中
2. 确认文件扩展名已添加到包含列表中
3. 查看日志输出，确认魔术数验证是否通过

### 问题：非压缩文件被误识别
**解决方案**：
1. 将文件扩展名添加到排除列表
2. 检查文件是否真的有压缩文件的魔术数

### 问题：想恢复默认设置
**解决方案**：
在设置对话框中将扩展名列表清空并保存，系统会使用默认值。
