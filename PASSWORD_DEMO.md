# 密码本管理功能演示

## 快速开始

### 1. 打开密码本管理

在主界面点击 **"编辑密码"** 按钮。

### 2. 添加一次性密码（临时测试）

场景：你有一个新下载的压缩包，想测试一个特定密码

**操作步骤**：
1. 在密码输入框输入：`测试密码123`
2. ✅ 勾选 **"一次性密码（仅本次运行有效）"**
3. 点击 **"添加密码"**

**效果**：
- 密码出现在列表**第一位**（序号1）
- 类型显示为 **"一次性"**
- 关闭软件后自动清除
- 不会影响永久密码本

### 3. 添加永久密码（常用密码）

场景：你发现了一个常用的密码，想保存下来

**操作步骤**：
1. 在密码输入框输入：`常用密码456`
2. ❌ **不勾选** "一次性密码"
3. 点击 **"添加密码"**

**效果**：
- 密码出现在列表末尾
- 类型显示为 **"永久"**
- 点击 **"保存"** 后会写入密码本文件
- 下次启动软件仍然有效

### 4. 密码使用次数自动排序（MRU）

**工作原理**：

```
初始状态：
序号  密码        类型   使用次数
1     密码A       永久   0
2     密码B       永久   0
3     密码C       永久   0

使用密码B解压了3个文件后：
序号  密码        类型   使用次数
1     密码B       永久   3     ← 自动排到第一位
2     密码A       永久   0
3     密码C       永久   0

再使用密码C解压了5个文件后：
序号  密码        类型   使用次数
1     密码C       永久   5     ← 排到第一位
2     密码B       永久   3     ← 降到第二位
3     密码A       永久   0
```

### 5. 混合使用示例

**场景**：添加一个临时密码测试

```
当前密码本：
序号  密码          类型      使用次数
1     常用密码A     永久      15
2     常用密码B     永久      8
3     很少用的C     永久      1

操作：添加一次性密码 "临时测试999"

结果：
序号  密码            类型      使用次数
1     临时测试999     一次性    0       ← 新添加，排在第一位
2     常用密码A       永久      15
3     常用密码B       永久      8
4     很少用的C       永久      1

测试顺序：临时测试999 → 常用密码A → 常用密码B → 很少用的C
```

### 6. 保存与清除

**点击"保存"按钮**：
- ✅ 永久密码按使用次数排序保存到文件
- ✅ 密码本文件路径被保存
- ✅ 下次启动自动加载

**关闭软件**：
- ❌ 一次性密码被清除
- ✅ 永久密码保留

## 实际使用流程

### 日常解压流程

```
1. 启动软件
   └─> 自动加载永久密码本（按使用次数排序）

2. 拖拽压缩包
   └─> 开始自动测试密码

3. 找到密码并解压
   └─> 该密码使用次数 +1
   └─> 下次自动排在更前面

4. 关闭软件
   └─> 保存使用次数到文件
   └─> 清除所有一次性密码
```

### 添加新密码流程

```
1. 点击"编辑密码"

2. 添加密码
   ├─> 勾选"一次性" → 仅本次运行有效 → 排在第一位
   └─> 不勾选 → 永久保存 → 排在末尾

3. 点击"保存"
   └─> 永久密码写入文件
   └─> 关闭对话框
```

## 提示与技巧

💡 **技巧1**：批量添加临时密码
- 可以连续添加多个一次性密码
- 它们会按添加顺序排列在前面
- 适合快速测试多个可能的密码

💡 **技巧2**：优化密码本
- 定期删除不再使用的密码
- 常用密码会自动浮到顶部
- 提高解压效率

💡 **技巧3**：备份密码本
- 密码本文件是普通 .txt 文件
- 可以复制备份到其他位置
- 也可以手动编辑（每行一个密码）

## 数据流向

```
添加一次性密码
  ↓
内存中的 OneTimePasswords 列表
  ↓
软件运行期间始终在最前面
  ↓
关闭软件时清除

添加永久密码
  ↓
内存中的 _permanentPasswords 列表
  ↓
按使用次数排序
  ↓
点击"保存"时写入 .txt 文件
  ↓
下次启动时从文件加载
```
