# 密码本管理功能说明

## 功能概述

本工具支持从配置文件（JSON）加载密码，不再依赖密码文本框。所有密码统一通过配置文件管理。

## 新增功能

### 1. 密码本导入功能

在"软件设置"对话框中，密码本文件路径选择区域新增了**"导入到配置"**按钮：

**操作步骤：**
1. 点击"浏览..."按钮选择密码本文件（TAB分隔的txt格式）
2. 点击**"导入到配置"**按钮
3. 密码将从文本文件导入到JSON配置文件中
4. 点击"保存配置"完成保存

**密码本文件格式：**
```
密码1		使用次数		最后使用时间(Ticks)
密码2		使用次数		最后使用时间(Ticks)
```

**说明：**
- 支持TAB分隔格式（两个Tab `\t\t`）
- 使用次数和时间为可选字段
- 导入的密码将按使用次数和最近使用时间自动排序（MRU优先）

### 2. 密码本导出功能

在"软件设置"对话框中，密码本文件路径选择区域新增了**"导出密码本"**按钮：

**操作步骤：**
1. 点击**"导出密码本"**按钮
2. 选择保存路径
3. 所有永久密码将导出为TAB分隔的文本文件

**导出格式：**
- 密码按综合评分排序（考虑使用次数和最近使用时间）
- 格式与导入格式相同，支持重新导入

### 3. 详细的密码测试日志

在主窗口日志中，现在会显示每个测试密码及其结果：

**日志示例：**
```
========== 开始测试: test.zip ==========
  ✗ 测试密码: 123456 -> 结果: 失败
  ✗ 测试密码: password -> 结果: 失败
  ✓ 测试密码: mypassword -> 结果: 成功
[test.zip] 密码测试成功，密码: mypassword
```

**说明：**
- ✓ 绿色文字表示密码正确
- ✗ 灰色文字表示密码错误
- 无密码时显示为"(无密码)"
- 如果测试过程出错，会显示红色错误信息

## 密码管理策略

### 密码加载顺序
1. **一次性密码**：本次运行中手动添加的临时密码（优先级最高）
2. **永久密码**：从配置文件加载的密码，按以下规则排序：
   - 使用次数多的优先
   - 最近使用过的优先
   - 新添加的密码优先

### 密码保存
- 所有密码保存在JSON配置文件中（位置：`%AppData%\AutoUnpackTool\settings.json`）
- 密码使用次数会自动记录
- 最后使用时间会自动更新

### 密码本文件路径
- 配置文件中的密码本路径（`PasswordFilePath`）仅用于记录原始导入来源
- 实际使用的密码都从JSON配置文件加载
- 可以通过导入功能更新密码

## 工作流程建议

### 首次使用
1. 如果有旧格式的密码本文件（txt），点击"导入到配置"
2. 点击"保存配置"
3. 后续无需再次导入，密码已保存在配置文件中

### 日常使用
1. 拖拽压缩包到主窗口
2. 程序自动从配置文件加载密码并测试
3. 日志会显示每个密码的测试结果
4. 找到正确密码后自动或手动解压（根据设置）

### 备份密码本
- 定期点击"导出密码本"备份密码
- 导出的文件可用于：
  - 在其他设备上导入
  - 作为密码本的备份
  - 查看密码使用统计

## 注意事项

1. **不再使用密码文本框**：主窗口中的密码文本已移除，所有密码从配置文件加载
2. **密码安全性**：密码以明文形式保存在JSON配置文件中，请妥善保管
3. **格式兼容**：导入功能支持多种格式：
   - 仅密码（每行一个）
   - 密码 + 使用次数（TAB分隔）
   - 密码 + 使用次数 + 时间（TAB分隔）
4. **去重处理**：导入时自动去除重复密码
5. **自动排序**：密码按综合评分自动排序，确保常用密码优先测试

## 故障排除

### 问题：密码测试总是失败
**解决方案：**
1. 检查7z.exe路径是否正确配置
2. 确认密码本中包含了正确的密码
3. 查看日志中的测试结果，确认密码已正确加载
4. 尝试手动添加密码（通过密码管理对话框）

### 问题：导入密码失败
**解决方案：**
1. 确认选择的文件格式正确（TAB分隔的txt文件）
2. 检查文件是否可读取（权限问题）
3. 查看文件内容是否符合格式要求

### 问题：找不到配置文件
**解决方案：**
1. 点击"打开文件夹"按钮查看配置文件位置
2. 首次使用会自动创建配置文件
3. 配置文件路径：`%AppData%\Roaming\AutoUnpackTool\settings.json`
